rocket_launch Quick Start

1

Create Your Project

Run the create command with your project name:

$ npm create walrus-app@latest my-walrus-app
2

Answer Interactive Prompts

The CLI will guide you through setup:

? Select SDK: @mysten/walrus
? Select framework: React + Vite
? Select use case: File Gallery
? Package manager: npm
3

Start Development

Navigate to your project and start the dev server:

$ cd my-walrus-app
$ npm run dev

layers Layer System

Walrus Starter Kit uses a modular 4-layer architecture designed for scalability and flexibility:

Layer 04: Use Case

APPLICATION

Pre-built templates: Simple Upload, File Gallery, DeFi/NFT integrations

Layer 03: Framework

UI LAYER

React, Vue, or Plain TypeScript with Vite for blazing-fast HMR

Layer 02: SDK Adapters

ABSTRACTION

Type-safe interfaces that abstract SDK-specific implementations

Layer 01: Base

FOUNDATION

Walrus Network and Sui blockchain infrastructure

extension Adapter Pattern

The adapter pattern allows you to switch between SDKs without changing your application code:

src/adapters/StorageAdapter.ts 
export interface StorageAdapter {
  // Upload a file and get a blob ID
  upload(file: File, options?: UploadOptions): Promise<string>;

  // Retrieve file metadata
  getInfo(id: string): Promise<BlobMetadata>;

  // Download blob content
  download(id: string): Promise<Blob>;

  // Remove from local tracking
  delete(id: string): Promise<void>;
}

Benefits

  • check_circle SDK-agnostic: Switch between @mysten/walrus, @tusky, or custom implementations
  • check_circle Type-safe: Full TypeScript support with compile-time guarantees
  • check_circle Testable: Easy to mock for unit tests and local development

help Troubleshooting

Installation fails with permission errors expand_more

Try running with sudo (Linux/Mac) or as Administrator (Windows), or use nvm/fnm to manage Node.js without requiring elevated permissions.

$ sudo npm create walrus-app@latest
Port 5173 is already in use expand_more

Vite will automatically try the next available port. You can also specify a custom port:

$ npm run dev -- --port 3000
TypeScript errors after installation expand_more

Make sure all dependencies are installed correctly:

$ rm -rf node_modules package-lock.json
$ npm install
Walrus SDK connection errors expand_more

Check your network configuration and ensure you're using the correct Walrus network (testnet/mainnet). Verify your aggregator/publisher endpoints in the config.
